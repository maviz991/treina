Fluxo de Trabalho: Publicação de Camadas e Metadados


Sobre o Fluxo de Trabalho

Para assegurar a qualidade, consistência e governança dos dados no Geohab, é crucial que cada etapa do processo de gestão seja bem definida. Este documento detalha o "Caminho Feliz" (Happy Path) para a publicação de dados, respeitando a separação técnica entre quem gerencia a estrutura (banco de dados) e quem produz o conteúdo (informação geográfica).


Definições de Papéis e Ambientes de Atuação

A tabela abaixo resume as responsabilidades no ciclo de vida do dado, indicando onde cada papel atua tecnicamente:

PAPEL AMBIENTE PRINCIPAL RESPONSABILIDADE E AÇÕES
Leitor
LEITOR_SETOR		 Visualizador / Catálogo
  • Consumo: Não participa da produção. Apenas visualiza dados aprovados.
Editor
EDITOR_SETOR		 Serviços (GeoServer)
via QGIS (WFS-T)
  • Produção de Conteúdo: Insere, desenha e edita feições (pontos, linhas, polígonos).
  • Manutenção: Pode apagar feições incorretas (conteúdo), mas nunca a camada (estrutura).
  • Documentação: Fornece as informações técnicas para a "Linhagem".
Revisor
REVISOR_SETOR		 Catálogo (GeoNetwork)
e Ferramentas de Validação
  • QA Geoespacial: Analisa a qualidade geométrica e topológica do trabalho do Editor.
  • Metadados: Cria e gerencia o registro de metadado (MGB 2.0).
  • Vínculo: Associa tecnicamente a Camada ao Metadado.
  • Aprovação: Autoriza a publicação.
Administrador Local
ADMIN_LOCAL_SETOR		 GeoServer (Admin) / GeoNetwork
  • Gestão de Estrutura: Cria tabelas vazias, configura Stores e Layers.
  • Publicação: Torna o serviço (WMS/WFS) visível para o público alvo.
  • Governança: Define permissões de acesso.

A Dupla Natureza do Revisor

O papel de Revisor possui uma função híbrida crucial:

  1. Técnica (Sistema): É quem opera o Catálogo para criar e editar metadados.
  2. Qualidade (Negócio): É quem valida se o dado geoespacial desenhado pelo Editor (no GeoServer/QGIS) está cartograficamente correto.

Fluxo de Trabalho Detalhado (11 Passos)

O processo ideal para levar um novo dado geoespacial à publicação envolve as seguintes etapas sequenciais:

1

Solicitação de Nova Estrutura (Demanda)

  • Agente: Administrador Local.
  • Ação: Identifica a necessidade de uma nova camada. Caso a tabela não exista no banco, formaliza a solicitação (ex: via chamado) detalhando campos, tipos de dados e projeção desejada.
2

Criação da Estrutura (Vazia)

  • Agente: Grupo Geo/Suporte ou Admin Local (com permissão).
  • Ambiente: Banco de Dados / GeoServer.
  • Ação: Cria a tabela física (estrutura) e configura o Layer no GeoServer. A camada é disponibilizada para o setor "em branco". Nenhum dado (feição) existe ainda.
3

Produção do Conteúdo Geoespacial

  • Agente: Editor.
  • Ambiente: QGIS conectado via WFS-T.
  • Ação:
    • Editor: Popula a camada. Desenha geometrias, insere registros e preenche atributos.
    • Estilo: Gera o arquivo de estilo (SLD) para a camada.
4

Submissão para Revisão

  • Agente: Editor.
  • Ação: Notifica o Revisor que a camada foi populada e está pronta para análise de qualidade.
5

Validação da Qualidade Geoespacial (QA)

  • Agente: Revisor.
  • Ação: Analisa a camada produzida quanto à precisão geométrica, topologia (ex: sobreposições) e completude de atributos.
    Nota: Se houver erros, devolve para o Passo 3 (Ciclo de Correção). Se aprovado, segue para o Passo 6.
6

Criação do Metadado

  • Agente: Revisor.
  • Ambiente: Catálogo (GeoNetwork).
  • Ação: Cria o registro de metadado (usando template MGB 2.0). Preenche identificação, sistema de referência e restrições.
7

Documentação da Linhagem

  • Agente: Editor (Fornece) e Revisor (Registra).
  • Ação: O Editor fornece os detalhes técnicos de como o dado foi produzido (processamento, fontes, datas) para que o Revisor preencha o campo "Linhagem" no metadado.
8

Associação Camada-Metadado

  • Agente: Revisor.
  • Ação: Realiza o vínculo técnico (Link) no GeoNetwork, associando o metadado ao serviço WMS/WFS correspondente no GeoServer.
9

Aprovação Final

  • Agente: Revisor.
  • Ação: Valida que o metadado está completo e aponta para a camada correta. Notifica o Administrador Local que o conjunto está pronto para publicação.
10

Publicação no Geohab

  • Agente: Administrador Local.
  • Ação:
    • No GeoServer: Verifica configurações finais e habilita a camada.
    • No GeoNetwork: Define as permissões de visualização (ex: Público ou Interno) e publica o metadado.
11

Disponibilização e Uso

A camada e seu metadado estão agora disponíveis no Geohab para consulta e uso pelos usuários com as devidas permissões, acessíveis através do Visualizador (MapStore 2) e do Catálogo.

Notas sobre o Fluxo

  • Flexibilidade: O Administrador Local pode executar tarefas de Editor ou Revisor se necessário, mas a separação de papéis é recomendada para boa governança.
  • Segurança: A definição de quem pode ver o dado (Público vs. Privado) é feita exclusivamente no Passo 10 pelo Administrador Local.

Conclusão

Este fluxo de trabalho estruturado garante que nenhum dado seja publicado sem passar por: 1) Criação estrutural adequada; 2) Produção de conteúdo qualificada; 3) Validação técnica e documental; 4) Publicação controlada. A clareza nas etapas evita a publicação de dados inconsistentes ou sem documentação.
