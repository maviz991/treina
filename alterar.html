Perfis de Acesso no Geohab


Sobre os Perfis de Acesso

No Geohab, a segurança da informação e o controle sobre os dados geoespaciais produzidos e gerenciados pela CDHU são primordiais. Para garantir que apenas as pessoas certas tenham acesso às informações e funcionalidades corretas, a plataforma utiliza um sistema robusto de perfis de acesso.

Este sistema permite delimitar com precisão os "poderes" e as funções de cada usuário. Para construir um perfil de acesso e definir o que um usuário pode fazer, o Geohab combina quatro elementos principais, que chamamos aqui de "instâncias de configuração":

Instâncias de Configuração

O sistema de perfis de acesso do Geohab é baseado em quatro elementos principais que trabalham em conjunto para definir as permissões dos usuários: Setores, Papéis, Usuários e Delegações.


Setores (Organizations)

As Organizations (Setores) no Geohab são usadas para agrupar usuários de forma lógica, geralmente refletindo a estrutura departamental ou de equipes da CDHU. Elas ajudam a organizar os usuários e podem ser usadas como um primeiro nível para agrupar permissões.

Como funcionam?

  • Cada usuário é obrigatoriamente associado a um Setor principal. E recebe o papel básico de cada setor, chamado de Leitor.
  • Um Super Usuário ou Administrador Local com privilégios adequados pode, se necessário, atribuir um papel de um setor a um usuário de outros Setores.
  • As Organizations (Setores) facilitam o gerenciamento, pois permitem aplicar certas configurações ou visibilidades de forma coletiva aos seus membros. Logo, os membros de cada setor tem acesso direto aos dados do seu setor.
  • No contexto da CDHU: Cada setor da companhia (Diretorias e/ou Superintendências) é tipicamente representado como uma organização dentro do Geohab.

Papéis (Roles)

Os Papéis são como "etiquetas" que definem conjuntos específicos de permissões. Um papel agrupa uma série de direitos sobre o que um usuário pode ou não pode fazer dentro da plataforma e em seus componentes, principalmente no Serviços (GeoServer) e no Catálogo (GeoNetwork), refletindo também no Visualizador (MapStore 2).

Como funcionam?

  • Os perfis de acesso são efetivamente definidos quando um Usuário é associado a um ou mais Papéis.
  • Esses papéis são armazenados no sistema de autenticação central do Geohab e são reconhecidos pelos diferentes componentes da plataforma.
  • Ao atribuir um papel a um Usuário, estamos, na prática, concedendo a ele um conjunto de permissões pré-definidas de acesso e manipulação de dados.
  • Todos os Setores têm quatro Papéis padrão: Administrador Local, Revisor, Editor e Leitor.

Distinção Técnica no Ecossistema (Banco de Dados e GeoServer)

Para compreender corretamente as permissões marcadas com asterisco (*) e (1) na tabela acima, é fundamental distinguir o acesso à ESTRUTURA do acesso ao CONTEÚDO dentro do Banco de Dados Geoespacial:

1. Gestão de Estrutura (Camada/Tabela)

Refere-se à criação (DDL) ou exclusão do "recipiente" dos dados.

  • Responsável: ADMIN_LOCAL
  • O que é: Criar uma nova tabela no banco ou publicar um novo Layer no GeoServer.
  • Editor: NÃO PODE criar ou destruir camadas.
2. Gestão de Conteúdo (Feições/Linhas)

Refere-se à manipulação (DML) dos dados dentro da camada.

  • Responsável: EDITOR
  • O que é: Desenhar (Insert), alterar (Update) ou apagar (Delete) geometrias e atributos.
  • Editor: PODE apagar feições individuais para corrigir erros.

Nota sobre o Revisor (1): Ele possui permissão de Criação (C) especificamente para Metadados no Catálogo (GeoNetwork), não para dados espaciais no GeoServer.

Granularidade de acesso

Os Papéis permitem um controle muito granular. Por exemplo, o Papel EDITOR_SETOR-A apenas poderá editar as camadas para as quais esse Papel foi especificamente delegado, no caso dados do Setor A.

É possível delegar Papéis de forma diferenciada para cada componente do Geohab (Serviços, Catálogo, Visualizador) e para diferentes níveis de recursos, como armazéns de dados, espaços geográficos, grupos de camadas, camadas individuais ou, em casos muito específicos, até mesmo linhas de dados dentro de uma camada.

- Em resumo: Os Papéis são a "ponte" que conecta os usuários e os setores e, consequentemente, às permissões, muitas vezes de forma contextualizada a recursos específicos.





Informação: Todos os usuário da plataforma têm, por padrão, o Papel de Usuário logado User. Isso garante que, no mínimo, todos possam visualizar os metadados e as camadas que foram publicadas para acesso geral (público da companhia), e não tenham acesso a dados privados.

Usuários (Users)

Os Usuários representam as contas individuais dos colaboradores, indivíduos ou entidades que precisam acessar o Geohab e seus recursos.

Como funcionam?

  • Cada Usuário possui um login e senha únicos (Login de rede CDHU).
  • Como mencionado, cada usuário é associado a pelo menos um Setor.
  • A um Usuário pode ser atribuído um ou mais Papéis, que podem ser papéis gerais da plataforma ou específicos para seu Setor ou para um determinado projeto ou recurso.

Delegações (Delegations)

Essa instância de configuração de acesso não é obrigatória, apenas usada em necessidades especificas. Então, as Delegações são o mecanismo pelo qual os Administradores Locais podem conceder direitos administrativos a um usuário de seu setor. Ou seja, dar direito a um usuário de seu setor a conceder acesso a dados do seu setor, no que compete a acesso de dados.

Como funcionam?

  • O Administrador Local pode conceder a um usuário poderes administrativos de acesso de dados a outro usuário "X" do seu setor, então, o usuário "x", poderá conceder ou remover acesso de dados a outros usuários de seu setor.

Delegação Padrão

Por padrão, no Geohab, os Administradores Locais são delegados a gerenciar todos os papéis de seu setor, então eles podem conceder ou remover acesso aos usuários de seu setor, ou de outro setor, em necessidades especiais.


Conclusão

A combinação dessas quatro instâncias – Setores, Papéis, Usuários e Delegações – formam o sistema de permissionamento do Geohab. Entender como elas interagem é crucial para administradores garantirem que os dados da CDHU sejam acessados e gerenciados de forma segura, eficiente e em conformidade com as políticas do Setor e da CDHU. Para os usuários, conhecer seu perfil de acesso ajuda a entender quais funcionalidades estão disponíveis e porquê.
