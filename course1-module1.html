Dados Geoespaciais

São informações que têm uma referência a uma localização na superfície da Terra. Em outras palavras, são dados que indicam onde algo está, além de descrever o que é e, às vezes, quando aconteceu.

Eles são a base de qualquer sistema de informação geográfica (GIS) e permitem analisar padrões, relações e tendências a partir de uma perspectiva espacial.

Fundamentalmente, temos quatro tipos de dados geoespaciais: Ponto, Linha, Polígono e Raster. No Goehab, quem serve esses dados no sistema é o componente Serviços (GeoServer).

Tipos Fundamentais:

Pontos
Representam uma única coordenada (ex: a localização de uma loja).
Linhas
Representam uma sequência de pontos conectados (ex: ruas, rios).
Polígonos
Representam uma área fechada (ex: bairros, florestas, lotes).
Raster
Representam uma grade de pixels, como uma fotografia (ex: imagens de satélite).

O que são Metadados?

Metadados são, basicamente, "dados sobre os dados". Eles são o contexto que nos ajuda a entender de onde os dados vieram, como foram coletados, com que precisão e quando. Sem essa "história", fica difícil confiar ou utilizar os dados da maneira correta.

Imagine um mapa com a localização de árvores. O dado são as coordenadas de cada árvore. O metadado informa se essas coordenadas foram coletadas por um especialista com um GNSS de alta precisão ou por um aplicativo de celular, e, crucialmente, em que ano isso foi feito . Se o ano de coleta de dados for antigo, pode nos levantar uma dúvida: será que as árvores ainda estão lá? Devemos cruzar esses dados com mais camadas para ter uma noção da situação da árera de estudo.

Esses detalhes são cruciais para a tomada de decisões. A ferramenta Catálogo (GeoNetwork) é a responsável por gerenciar os metadados no Geohab.

Nota: Metadados são o que trazem confiança, clareza e utilidade aos dados geoespaciais. Sem eles, os dados perdem seu contexto e, muitas vezes, seu valor.

Formatos de Arquivos Geoespaciais

A escolha do formato de arquivo correto é crucial para garantir a integridade, o desempenho e a compatibilidade dos seus dados. Os dados geoespaciais são divididos em duas categorias principais: vetoriais (pontos, linhas e polígonos) e matriciais ou raster (grades de pixels, como imagens).

Abaixo, apresentamos os formatos mais comuns e as recomendações oficiais da CDHU para o armazenamento e compartilhamento de dados.

×
Diferentes formatos de arquivo são usados para dados vetoriais e matriciais.

Formatos Vetoriais (Pontos, Linhas, Polígonos)

Shapefile (.shp)
Status: Obsoleto - Evitar uso.
- Embora muito popular, é um formato antigo composto por múltiplos arquivos (.shp, .shx, .dbf, etc.), propenso a erros e com limitações (ex: nomes de colunas com 10 caracteres). Seu uso deve ser evitado.
GeoPackage (.gpkg)
Status: Recomendado.
- É o sucessor moderno do Shapefile. Consiste em um único arquivo (.gpkg) baseado em SQLite, robusto, flexível e sem as limitações do formato antigo. É o padrão aberto para troca de dados geoespaciais.
GeoJSON / KML
Status: Uso Específico.
- Formatos baseados em texto, ideais para aplicações web (GeoJSON) e visualização em plataformas como o Google Earth (KML). São ótimos para compartilhamento e visualização, mas menos robustos para análise complexa.

Formatos Matriciais (Raster)

GeoTIFF (.tif / .tiff)
Status: Padrão da Indústria.
- É o formato mais comum para imagens georreferenciadas, como ortofotos e modelos de elevação. É um arquivo de imagem padrão (TIFF) que contém metadados geográficos adicionais (geotags).
COG - Cloud Optimized GeoTIFF (.tif / .tiff)
Status: Padrão moderno para dados em nuvem.
- É uma variação do GeoTIFF projetada para acesso eficiente via web e ambientes cloud. Organiza os dados internamente em tiles e overviews, permitindo a leitura parcial do arquivo sem necessidade de download completo. É amplamente utilizado em geoserviços e aplicações web para visualização e análise rápida de grandes rasters georreferenciados. Este é o formato recomendado para o Geohab.

A Abordagem Moderna: Bancos de Dados

Embora arquivos como o GeoPackage sejam excelentes para troca e armazenamento local, a forma mais robusta e escalável de gerenciar dados geoespaciais é através de um banco de dados espacial, como o PostgreSQL com a extensão PostGIS. Isso centraliza a informação, garante segurança, permite acesso multiusuário e otimiza consultas complexas, sendo a base de plataformas como o Geohab.

Além disso, seu setor ou grupo de trabalho sempre está sincronizado com a última versão do dado. Com isso se estabelece uma única fonte de verdade, sobre controle e dominío da alta gestão do setor, na figura do Administrador Local.

Nota: Padrão CDHU - Ao salvar ou exportar dados geoespaciais para arquivos locais, o formato recomendado e padrão a ser utilizado é o GeoPackage (.gpkg). Mas, ele deve ser usado como intermediário ao banco de dados, o destino de armazenamento final é o banco de dados.


Lembre-se: A fonte de verdade dos dados geoespaciais da CDHU deve ser o Banco de dados!

Padrões OGC: Interoperabilidade

Interoperabilidade, através de Padrões, é a capacidade de diferentes sistemas e organizações trabalharem em conjunto. No contexto do Geohab, isso significa que diversas ferramentas, programas e formatos de dados podem "falar a mesma língua", permitindo a troca de informações de forma fluida e eficiente.

O objetivo é eliminar barreiras técnicas, como formatos incompatíveis, garantindo que todos na CDHU tenham acesso aos dados mais atualizados, economizando tempo e evitando erros.

Noções de Rede

Para usar o Geohab eficientemente, é útil entender alguns conceitos básicos sobre redes de computadores e a internet. Pense na internet como uma vasta rede de estradas digitais, onde os dados viajam constantemente de um lugar para outro, permitindo que sistemas geográficos compartilhem informações em tempo real.

Cada "lugar" nesta rede tem um endereço único, chamado URL (Uniform Resource Locator), que funciona como o endereço de um site. Quando você acessa o Geohab, seu computador e os servidores da CDHU trocam mensagens usando regras chamadas HTTP (Hypertext Transfer Protocol), que estabelecem como essa comunicação deve ocorrer.

Geoserviços

Geoserviços são serviços web especializados que disponibilizam dados geográficos e funcionalidades de processamento através da internet. Eles permitem que diferentes aplicações acessem, visualizem e analisem dados espaciais sem precisar baixar arquivos ou instalar softwares especializados.

No contexto do Geohab, os geoserviços são a tecnologia que torna possível acessar mapas e dados geográficos diretamente no navegador, permitindo análise em tempo real e integração com outros sistemas.

1:40 min Vídeo

Principais Tipos de Geoserviços

WMS (Web Map Service)
Fornece imagens de mapas como imagens estáticas (PNG, JPEG). Ideal para visualização, mas não permite análise dos dados brutos. O Geohab utiliza WMS para exibir camadas base e de referência.
WFS (Web Feature Service)
Fornece os dados vetoriais brutos (pontos, linhas, polígonos) em formato GML. Permite análise espacial, consultas e edição dos dados. Usado no Geohab para camadas que necessitam interação.
WCS (Web Coverage Service)
Fornece dados matriciais ou de cobertura (imagens de satélite, modelos de elevação). Permite acesso aos valores brutos dos pixels para análises especializadas.

Vantagens dos Geoserviços no Geohab

A adoção de geoserviços traz múltiplos benefícios para a gestão de dados geográficos na CDHU:

1 Acesso Centralizado

Dados geográficos ficam armazenados em servidores centralizados, garantindo que todos os usuários acessem sempre a versão mais atualizada e eliminando problemas com múltiplas cópias desatualizadass.

2 Integração entre Sistemas

Diferentes sistemas e aplicações podem consumir os mesmos dados através dos geoserviços, garantindo consistência e evitando retrabalho na atualização de informações em múltiplas plataformas.

3 Análise em Tempo Real

Os dados podem ser analisados diretamente no servidor, permitindo processamentos complexos sem necessidade de transferir grandes volumes de dados para o computador do usuário.

Perfis de Acesso

Os perfis de acesso no Geohab são a chave para garantir a segurança e a organização dos dados geoespaciais da CDHU. Através de um sistema de permissões personalizadas, o Geohab controla o acesso às informações, definindo quem pode visualizar, editar ou gerenciar os dados.

Cada perfil (Usuário, Leitor, Editor, Revisor, Administrador Local e Super Usuário) possui responsabilidades e permissões específicas, assegurando que as informações estejam sempre protegidas e disponíveis para quem realmente precisa. Com os perfis de acesso, o Geohab promove a colaboração entre os setores, evita o acesso não autorizado e garante a integridade e a confiabilidade dos dados geoespaciais da Companhia.

Ícone representando segurança e controle de acesso

Níveis de Permissão

Perfis de Acesso Disponíveis:

Usuário Leitor Editor Revisor Administrador Local Super Usuário

Descrição Detalhada:

Usuário
Acesso apenas para visualização de dados e mapas, sem permissão para alterações. E apenas dados publicos não vê dados privados de nenhum setor.
Leitor
Acesso apenas para visualização de dados e mapas, sem permissão para alterações. Apenas para leitura de dados de um Setor.
Editor
Permite visualizar e editar informações geoespaciais, mas não publicar alterações.
Revisor
Além de editar, pode revisar e aprovar alterações feitas por outros usuários.
Administrador Local
Gerencia usuários e permissões dentro de sua unidade ou departamento.
Super Usuário
Acesso completo ao sistema, com permissões para gerenciar todos os aspectos do Geohab.

Nota Essencial: O correto gerenciamento de perfis de acesso é fundamental para garantir a segurança, integridade e confiabilidade dos dados geoespaciais da CDHU.
