1

Como os Mapas "Ganham Vida" na sua Tela?

Visualizador (MapStore 2)
Interface que você vê e interage no Geohab, construída com bibliotecas JavaScript avançadas para criar uma experiência de mapa intuitiva.
Serviços (GeoServer)
Servidor que armazena e fornece os dados geoespaciais para o visualizador, respondendo às requisições e enviando as informações no formato adequado.
Bibliotecas JavaScript (OpenLayers)
Conjunto de ferramentas de programação que permitem "desenhar" mapas, conectar com servidores e criar interatividade na tela do navegador.
2

KML: A Linguagem dos Lugares

KML (Keyhole Markup Language)
Formato de arquivo baseado em XML que descreve pontos, linhas, polígonos e imagens para serem exibidos em mapas. Popularizado pelo Google Earth.
GeoJSON
Formato aberto baseado em JSON para codificar estruturas de dados geográficas. Ideal para aplicações web e interoperabilidade entre sistemas.
Shapefile
Formato vetorial clássico de dados geoespaciais, composto por múltiplos arquivos (.shp, .shx, .dbf). Embora antigo, ainda é amplamente utilizado.
3

Bibliotecas de Mapas: As Ferramentas que "Desenham"

OpenLayers
Biblioteca JavaScript de código aberto para exibir mapas em navegadores. Permite criar camadas, adicionar controles de interação e conectar com diversos serviços de mapas.
Leaflet
Biblioteca JavaScript leve e simples para mapas interativos. Focada em simplicidade, desempenho e usabilidade em diversos dispositivos.
MapStore 2
Plataforma completa de visualização de mapas utilizada no Geohab. Oferece uma interface amigável que permite criar, personalizar e compartilhar mapas interativos sem necessidade de conhecimento técnico avançado.
4

Como isso se Conecta no Ecossistema do Geohab?

Visualização
O Visualizador do Geohab renderiza os dados na tela, permitindo que você visualize e interaja com as camadas de informação.
Requisições
Quando você navega pelo mapa, o visualizador faz requisições ao GeoServer para obter os dados necessários para exibir na área visível.
Processamento
Os dados recebidos (imagens, KML, GeoJSON, etc.) são interpretados e processados pela biblioteca JavaScript antes de serem desenhados na tela do navegador.

O que você precisa saber?

Como usuário do Geohab, você não precisa saber programar ou criar KML manualmente. O importante é entender que existe um ecossistema de tecnologias trabalhando juntas para que você possa visualizar e interagir com os dados geoespaciais de maneira intuitiva!

Criando Mapas no Geohab

A funcionalidade de 'Criar Mapas' do Geohab permite criar mapas interativos a partir de camadas geoespaciais. Você pode configurar estilos, adicionar legendas, ativar pop-ups, controlar a visibilidade das camadas e compartilhar o mapa com outros usuários ou incorporá-lo em websites.

Pré-requisitos: Ter um usuário autenticado no Geohab com permissão para criar mapas e acesso a camadas públicas ou privadas (WMS, WFS, GeoJSON, etc.).

  • Use transições suaves entre as seções para criar um fluxo narrativo contínuo.
  • Insira dashboards via página de web. Copie o URL do Dashboard criado na plataforma e cole no GeoStory.
  • Ideal para apresentações públicas, relatórios de projetos ou material educacional.
  • Mantenha os textos curtos e objetivos em cada seção.
  • Crie uma estrutura lógica: introdução → contexto → análise → conclusão.
  • Use imagens em boa resolução para um visual profissional.
  • Teste a sua GeoStory em diferentes tamanhos de tela para garantir a responsividade.
