Tecnologias por Trás dos Mapas


Já se perguntou como os mapas interativos na internet funcionam? Vamos descobrir as tecnologias que fazem a mágica acontecer!

A capacidade de visualizar dados geoespaciais em navegadores web representa uma revolução na forma como interagimos com informações espaciais. Essa experiência é possível graças a um ecossistema de tecnologias que trabalham em conjunto, desde servidores especializados até bibliotecas de programação que "desenham" mapas diretamente na tela do seu dispositivo.

No Geohab, essas tecnologias foram integradas para criar uma plataforma acessível que permite explorar dados complexos sem necessidade de conhecimento técnico avançado.

Nesta seção, vamos explorar de forma simples e prática as tecnologias que permitem que você visualize e interaja com mapas no Geohab.

Como os Mapas "Ganham Vida" na sua Tela?

O processo de exibição de mapas interativos envolve múltiplas camadas tecnológicas que trabalham em harmonia para transformar dados brutos em visualizações dinâmicas e responsivas.

Visualizador (MapStore 2)
Interface que você vê e interage no Geohab, construída com bibliotecas JavaScript avançadas para criar uma experiência de mapa intuitiva. Oferece controles de navegação, ferramentas de consulta e personalização de camadas.
Serviços (GeoServer)
Servidor que armazena e fornece os dados geoespaciais para o visualizador, respondendo às requisições e enviando as informações no formato adequado. Implementa padrões OGC como WMS, WFS e WMTS.
Bibliotecas JavaScript (OpenLayers)
Conjunto de ferramentas de programação que permitem "desenhar" mapas, conectar com servidores e criar interatividade na tela do navegador. OpenLayers é uma das bibliotecas mais robustas para aplicações empresariais.

KML: A Linguagem dos Lugares

Para que dados geoespaciais possam ser compartilhados e visualizados em diferentes plataformas, é necessário utilizar formatos padronizados que descrevam tanto a geometria quanto os atributos associados.

KML (Keyhole Markup Language)
Formato de arquivo baseado em XML que descreve pontos, linhas, polígonos e imagens para serem exibidos em mapas. Popularizado pelo Google Earth, permite incluir estilos, descrições e até mesmo tours animados.
GeoJSON
Formato aberto baseado em JSON para codificar estruturas de dados geográficas. Ideal para aplicações web e interoperabilidade entre sistemas devido à sua compatibilidade nativa com JavaScript e APIs REST.
Shapefile
Formato vetorial clássico de dados geoespaciais, composto por múltiplos arquivos (.shp, .shx, .dbf). Embora antigo, ainda é amplamente utilizado devido à sua compatibilidade com a maioria dos softwares GIS.

Bibliotecas de Mapas: As Ferramentas que "Desenham"

As bibliotecas JavaScript são o motor por trás dos mapas interativos na web, fornecendo as funcionalidades necessárias para renderizar, manipular e interagir com dados geoespaciais diretamente no navegador.

OpenLayers
Biblioteca JavaScript de código aberto para exibir mapas em navegadores. Permite criar camadas, adicionar controles de interação e conectar com diversos serviços de mapas. É a biblioteca utilizada pelo MapStore 2.
Leaflet
Biblioteca JavaScript leve e simples para mapas interativos. Focada em simplicidade, desempenho e usabilidade em diversos dispositivos. Popular para aplicações web que necessitam de visualização básica de mapas.
MapStore 2
Plataforma completa de visualização de mapas utilizada no Geohab. Oferece uma interface amigável que permite criar, personalizar e compartilhar mapas interativos sem necessidade de conhecimento técnico avançado.

Como isso se Conecta no Ecossistema do Geohab?

No Geohab, essas tecnologias foram integradas para criar um fluxo contínuo de dados, desde o armazenamento até a visualização, garantindo performance e usabilidade para os usuários finais.

Visualização
O Visualizador do Geohab renderiza os dados na tela, permitindo que você visualize e interaja com as camadas de informação através de controles intuitivos de navegação e consulta.
Requisições
Quando você navega pelo mapa, o visualizador faz requisições ao GeoServer para obter os dados necessários para exibir na área visível, otimizando o tráfego de rede.
Processamento
Os dados recebidos (imagens, KML, GeoJSON, etc.) são interpretados e processados pela biblioteca JavaScript antes de serem desenhados na tela do navegador, aplicando estilos e simbologias definidos.

O que você precisa saber?

Como usuário do Geohab, você não precisa saber programar ou criar KML manualmente. O importante é entender que existe um ecossistema de tecnologias trabalhando juntas para que você possa visualizar e interagir com os dados geoespaciais de maneira intuitiva!

Criando Mapas no Geohab


A funcionalidade de 'Criar Mapas' do Geohab permite criar mapas interativos a partir de camadas geoespaciais. Você pode configurar estilos, adicionar legendas, ativar pop-ups, controlar a visibilidade das camadas e compartilhar o mapa com outros usuários ou incorporá-lo em websites.

Pré-requisitos: Ter um usuário autenticado no Geohab com permissão para criar mapas e acesso a camadas públicas ou privadas (WMS, WFS, GeoJSON, etc.).

1. Criando um Novo Mapa

No menu principal, clique em Apps e depois em Criar novo mapa. O sistema abrirá o editor de mapas em branco.

  • Menu Apps: Acesso às ferramentas do Geohab.
  • Criar novo mapa: Inicia um novo mapa em branco.

2. Adicionando Camadas ao Mapa

Clique no botão Adicionar Camada (ícone de camada no canto superior direito) e escolha a origem:

  • Adicionar Camada: Abre a janela para adicionar novas camadas.
  • Catálogo de Camadas: Busca camadas no Geohab.
  • Pesquisa: Digite o nome da camada desejada.
  • Adicionar ao Mapa: Adiciona a camada selecionada.

3. Personalizando o Mapa

Ative ou desative camadas na lista lateral. Clique em uma camada para abrir o menu de configurações.

  • Ativar/Desativar: Mostra ou oculta a camada no mapa.
  • Configurações: Abre o menu de opções da camada.
  • Estilo: Personaliza cores, símbolos e legendas.

4. Configurações de Estilo

No menu de configurações, você pode personalizar:

  • Aba de Estilo: Configurações gerais de aparência.
  • Preenchimento: Estilo para áreas poligonais.
  • Ícones: Estilo para pontos e marcadores.
  • Legendas: Configuração de textos no mapa.

5. Exemplo de Preenchimento Categorizado

Clique em regras de preenchimento e selecione o modo Categorizado. Defina um campo (ex: 'area_km2') para criar categorias de cor a partir dos valores desse campo.

  • Regras de preenchimento: Abre as opções de estilo.
  • Categorizado: Cria cores por categorias.
  • Campo area_km2: Base para as categorias.
  • Aplicar: Salva as configurações.

6. Configurando o Fundo do Mapa

Clique em Fundo ou Base Map na barra lateral. Escolha entre fundo padrão, satélite, OpenStreetMap, ou adicione serviços personalizados (TMS, XYZ).

  • Base Map: Abre as opções de fundo.
  • Satélite: Fundo com imagens de satélite.
  • Serviço personalizado: Adiciona outros fundos.

7. Salvando o Mapa

Clique em Salvar na barra lateral e preencha:

  • Salvar: Abre a janela para salvar o mapa.
  • Título: Nome do seu mapa.
  • Descrição/Tags: Detalhes e palavras-chave.
  • Detalhes: Adicionar informações adicionais.
  • Permissões: Definir quem pode acessar.
  • Salvar final: Confirma e salva o mapa.

Após salvar, use o botão Compartilhar. Opções disponíveis:

  • Compartilhar: Abre as opções de compartilhamento.
  • Link direto: URL para acessar o mapa.
  • Incorporar iframe: Código para websites.

Dicas e Boas Práticas

  • Dênomes significativos às camadas.
  • Agrupe camadas por tema ou função.
  • Teste o mapa em resoluções diferentes.
  • Mantenha a interface limpa e compreensível para usuários finais.
  • Use mapas salvos como base para Dashboards e GeoStories.
